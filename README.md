# ROS 2 Learning Project

## Project Overview

This project focuses on learning the fundamentals of **ROS 2 (Robot Operating System 2)** and documenting the learning process in a clear and practical way.

The purpose of the project is to build a basic understanding of how ROS 2 is used in robotics development. The documentation is also intended to help future students understand the first steps of working with ROS 2.

The project starts from the basics and gradually moves toward creating simple ROS 2 applications, using visualization and simulation tools, and testing communication between multiple devices.

---

## Project Goals

The main goals of this project are to:

- Understand what ROS 2 is and how it is used in robotics
- Set up a working ROS 2 development environment
- Learn the main ROS 2 concepts, such as nodes, topics, services and launch files
- Create simple custom ROS 2 nodes
- Use ROS 2 visualization and simulation tools
- Test ROS 2 communication between different devices
- Document the learning process, problems and solutions for future use

---

## Project Phases

### 1. Development Environment

The first phase is to set up the required development environment. This includes installing Linux, ROS 2 and the necessary development tools.

### 2. ROS 2 Fundamentals

The second phase focuses on learning the basic concepts of ROS 2, including nodes, topics, publishers, subscribers, services, parameters, packages and launch files.

### 3. First Practical Exercises

The third phase includes simple ROS 2 exercises and demonstrations. The goal is to understand how ROS 2 systems work in practice.

### 4. Custom ROS 2 Nodes

The fourth phase focuses on creating simple custom ROS 2 nodes. These nodes can be used to send, receive and process data inside a ROS 2 system.

### 5. Visualization and Simulation

The fifth phase introduces tools such as RViz and Gazebo. These tools are used to visualize robot data and test robotics concepts in a simulated environment.

### 6. Distributed ROS 2 Communication

The final phase focuses on testing ROS 2 communication between multiple devices, such as a Linux laptop and a Raspberry Pi. This helps to understand how ROS 2 can be used in a real multi-device robotics system.

## Expected Outcome

By the end of this project, the goal is to have a working ROS 2 development environment, basic custom ROS 2 applications, practical experience with visualization and simulation tools, and clear documentation that can be used by other students starting with ROS 2.

The main focus of the project is not only to achieve working examples, but also to document the learning path in a way that is understandable, repeatable and useful for future projects.
